# Tech Debt Tracker

## 현재 기술 부채 목록

### [높음] TD-001: 핵심 비즈니스 로직 단위 테스트 부재

- **영향**: saju.ts, ten_gods.ts, gyeok_guk.ts, yong_sin.ts 등 핵심 모듈에 단위 테스트 없음
- **위험**: 리팩토링/기능 추가 시 회귀 버그 감지 불가
- **현재**: 검증 테스트 3개만 존재 (jdn, kasi-anchor, solar-terms)
- **해결**: 최소 saju.ts 10단계 파이프라인 각 단계별 단위 테스트 추가

### [높음] TD-002: fortune.ts 대형 모듈 (715줄)

- **영향**: 운세 해석 로직이 단일 파일에 집중
- **위험**: 유지보수 어려움, 변경 시 부작용 범위 넓음
- **해결**: 운세 유형별 (general, career, wealth, health, love) 서브모듈 분할

### [중간] TD-003: tool-handler.ts의 `unknown` 캐스팅

- **영향**: `args as Parameters<typeof handler>[0]` 패턴으로 타입 안전성 약화
- **위험**: 런타임 타입 불일치 시 디버깅 어려움
- **해결**: Zod 스키마 parse를 핸들러 진입 시점에 적용

### [중간] TD-004: saju.ts 대형 모듈 (657줄)

- **영향**: 10단계 파이프라인 + 보조 함수가 단일 파일에 집중
- **위험**: 파이프라인 단계 간 경계 모호, 테스트 어려움
- **해결**: 파이프라인 단계별 함수 분리 (파일은 유지하되 함수 수준 정리)

### [중간] TD-005: 통합 테스트 부재

- **영향**: MCP 도구 호출 -> 결과 검증 경로 미테스트
- **위험**: 도구 스키마와 핸들러 불일치 감지 불가
- **해결**: 7개 도구 각각의 기본 호출 통합 테스트 추가

### [낮음] TD-006: 일부 모듈 방어 코드 부족

- **영향**: 경계 조건(범위 외 날짜, 잘못된 도시명 등) 처리 불완전
- **위험**: 예외적 입력 시 불명확한 에러
- **해결**: 각 모듈 입력 검증 강화
