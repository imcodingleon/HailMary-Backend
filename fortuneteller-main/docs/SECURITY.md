# Security

> fortuneteller MCP 서버 보안 고려사항

## 위협 모델

이 프로젝트는 사주 분석 MCP 서버로, 민감한 개인정보(생년월일시, 성별, 출생지)를 처리한다.

### 처리하는 민감 데이터

| 데이터 | 민감도 | 저장 여부 |
|--------|--------|-----------|
| 생년월일시 | 중 | 미저장 (메모리 캐시만, 세션 종료 시 소멸) |
| 성별 | 저 | 미저장 |
| 출생지 | 저 | 미저장 |
| 이름 | 중 | 미처리 (도구에 전달하지 않음) |

## 입력 검증

### Zod 스키마 검증

모든 MCP 도구 입력은 `src/schemas/index.ts`의 Zod 스키마로 검증된다:

- 날짜 형식: `YYYY-MM-DD` 패턴 매칭
- 시간 형식: `HH:mm` 패턴 매칭
- 달력 타입: `'solar' | 'lunar'` 리터럴
- 성별: `'male' | 'female'` 리터럴
- 숫자 범위: 월 1-12, 시 0-23

### 문자열 처리

- 입력 문자열은 구조화된 포맷만 허용 (자유 텍스트 입력 없음)
- SQL 인젝션 해당 없음 (DB 미사용)
- XSS 해당 없음 (HTML 렌더링 없음)

## 데이터 보호

### 저장하지 않는 것

- 사용자 개인정보를 파일시스템, DB, 외부 서비스에 저장하지 않음
- 로그에 개인정보를 기록하지 않음
- 외부 API로 개인정보를 전송하지 않음

### 메모리 캐시

- `performance_cache.ts`: 계산 결과를 메모리에 캐싱
- 서버 프로세스 종료 시 모든 캐시 소멸
- 캐시 키에 개인 식별 가능 정보 포함 (birthDate, birthTime, gender)
- **개선 필요**: 캐시 TTL 미설정 -> 장기 실행 서버에서 무한 성장 가능

## 네트워크 보안

### 외부 통신

- 외부 API 호출: **없음**
- 외부 데이터 수신: **없음**
- 모든 데이터 로컬 테이블 기반

### HTTP 서버 (server-http.ts)

- CORS: `*` (모든 출처 허용) -> 프로덕션 배포 시 제한 검토
- SSE 세션 관리: sessionId 기반
- 인증: 미구현 (MCP 프로토콜 수준에서 처리 위임)

## 의존성 보안

### 직접 의존성 (4개)

| 패키지 | 위험도 | 비고 |
|--------|--------|------|
| `@modelcontextprotocol/sdk` | 저 | Anthropic 공식 |
| `@smithery/sdk` | 저 | 배포 통합 |
| `date-fns` / `date-fns-tz` | 저 | 널리 사용되는 날짜 라이브러리 |

### 권장 사항

- 정기적으로 `npm audit` 실행
- 의존성 업데이트 시 `npm test` + `npm run build` 확인
