# Core Beliefs

> fortuneteller 프로젝트의 핵심 설계 신념

## 1. 오프라인 우선 (Offline-First)

모든 사주 계산 데이터는 로컬 테이블(1900-2200, 300년)에 내장한다.
외부 API 의존성을 절대 추가하지 않는다.

**근거**: 사주 분석은 정확성과 일관성이 핵심. 외부 API 장애, 응답 지연, 데이터 변경으로 인한 계산 결과 불일치를 원천 차단한다.

## 2. 파이프라인 불가침 (Pipeline Immutability)

10단계 사주 계산 파이프라인의 순서를 변경하거나 단계를 건너뛸 수 없다. 각 단계는 이전 단계 결과에 의존하며, 이 의존 체인은 명리학 이론에 기반한다.

```
음력변환 -> 진태양시 -> 4기둥 -> 지장간 -> 오행 -> 십성 -> 신살 -> 강약 -> 격국 -> 용신
```

## 3. 레이어 방향성 (Layer Directionality)

의존성은 상위 -> 하위 방향만 허용한다. 역방향 참조와 순환 참조를 절대 금지한다. 이 규칙은 코드의 테스트 가능성, 이해 용이성, 변경 안전성을 보장한다.

## 4. 타입 안전성 우선 (Type Safety First)

TypeScript strict 모드를 활용하며, 리터럴 유니온 타입(천간 10개, 지지 12개, 오행 5개 등)으로 도메인 불변량을 컴파일 타임에 보장한다.

## 5. 한국어 중심 (Korean-First)

- 에러 메시지는 한국어로 작성하며 올바른 형식을 안내한다
- 사주 용어는 한국어 우선, 필요시 한자/영문 병기
- 모든 에이전트 응답은 한국어

## 6. 데이터 무결성 보호 (Data Integrity)

절기/음력 데이터 테이블(`solar_terms_*.ts`, `lunar_table_*.ts`)은 수동 편집을 금지한다. 이 테이블은 천문 관측 데이터 기반이며, 수동 편집은 계산 정확도 전체에 영향을 미친다.

## 7. 에러 격리 (Error Isolation)

개별 MCP 도구 호출 실패가 서버 전체에 영향을 주지 않는다. try-catch + isError 플래그로 격리하며, 한국어 에러 메시지를 반환한다.

## 8. 유파 중립성 (School Neutrality)

5개 명리 해석 유파(자평명리, 현대명리, 적천수, 궁통보감, 신살중심)를 동등하게 지원한다. 특정 유파를 우대하거나 배제하지 않으며, 사용자가 `manage_settings`로 선택할 수 있다.
